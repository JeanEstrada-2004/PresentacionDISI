# Evidencias seleccionadas para presentación - NANU TECH DISI

Este paquete contiene una selección reducida de imágenes y documentos de referencia para construir la presentación final del trabajo DISI.

## Criterio de selección

Se eligieron evidencias que ayudan a defender el ciclo de vida del sistema: Canvas, arquitectura, roles/canales, trazabilidad, datos, web, app/UX, QA y despliegue preparado.

No se incluyeron como evidencias principales capturas que puedan confundir el MVP con funcionalidades excluidas, como GPS en tiempo real, tracking, combustible, facturación, push notifications o producción AWS garantizada.

## Uso recomendado por diapositiva

1. Portada: no requiere evidencia.
2. Contexto/problema: usar texto y/o diagrama de roles si se desea.
3. Idea TIC: `01_canvas/01_lean_canvas_nanutech.png`.
4. Comparación de mercado: construir tabla en HTML; no requiere imagen.
5. Alcance MVP: usar tabla en HTML o `02_diagramas_clave/04_trazabilidad_resumida_mvp.png`.
6. Actores y canales: `02_diagramas_clave/02_vista_roles_modulos.png`.
7. SDLC/requerimientos/trazabilidad: `02_diagramas_clave/04_trazabilidad_resumida_mvp.png`.
8. Diseño UX/Figma: usar 2 o 3 imágenes de `07_diseno_ux_referencia/` y agregar botón a Figma.
9. Arquitectura: `02_diagramas_clave/01_arquitectura_general_mvp.png`.
10. DER: `03_modelo_datos/01_der_postgresql_central.png`.
11. Implementación web: usar capturas seleccionadas de admin, gerente y chofer.
12. App móvil/offline: usar `02_diagramas_clave/03_sync_offline_basica.png` y, si existen capturas reales externas de app, usarlas junto al link de evidencias.
13. Backend/API: `02_diagramas_clave/05_mapa_rol_modulo_api_dato.png`.
14. Pruebas: `08_pruebas_qa/01_collage_evidencias_qa.png`.
15. Despliegue/costos: `02_diagramas_clave/06_arquitectura_aws_preparada.png`.
16. Cierre/resultados: usar combinación o resumen visual de arquitectura + web + app + QA.

## Reglas para la presentación

- Rotular las imágenes UX como diseño/prototipo, no como captura implementada.
- Rotular AWS como preparación para cloud/testing, no producción garantizada.
- Presentar el chofer móvil como canal principal y el chofer web como canal secundario.
- Presentar offline como soporte básico controlado, no sincronización productiva robusta.
- Usar enlaces para Figma, sistema web, evidencias app móvil y anexos, en vez de saturar la presentación.

Ver también `00_MANIFIESTO_EVIDENCIAS_PRESENTACION.csv` para el detalle de cada archivo.
