# Informe final de presentación - NANU TECH

## 1. Cambios aplicados en diapositivas 25 a 28

### Diapositiva 25: Pruebas y validaciones

- Se reorganizó la diapositiva para explicar qué se validó, no solo mostrar el collage QA.
- Se agregaron tarjetas de pruebas funcionales, integración, roles/permisos, app móvil y offline/sync.
- El collage QA se mantiene como evidencia visual general, con leyenda interpretativa.
- Se incorporó una frase de defensa que delimita el alcance: las pruebas validan el funcionamiento base del MVP, no una operación productiva avanzada.

### Diapositiva 26: Despliegue, AWS y costos

- Se aclaró que AWS está preparado para local/testing/cloud inicial, no para producción empresarial garantizada.
- Se mantuvo la arquitectura AWS como evidencia y se agregaron chips de servicios considerados.
- Se mantuvo el total de S/ 17,088 y se agregó desglose referencial:
  - Desarrollo y adecuación del MVP: S/ 9,600.
  - Infraestructura cloud/testing inicial: S/ 1,488.
  - Capacitación por rol: S/ 1,200.
  - Soporte y mantenimiento inicial: S/ 3,600.
  - Contingencia técnica: S/ 1,200.
- Se aclaró que el costo es académico y referencial, no cotización comercial.

### Diapositiva 27: Operación, capacitación y mantenimiento

- Se separó la explicación en cuatro bloques: puesta en marcha, capacitación por rol, mantenimiento y evolución futura.
- Se detalló capacitación por rol: administrador 8 h, gerente 4 h y chofer 4 h prácticas.
- Se incluyeron tipos de mantenimiento: correctivo, adaptativo, perfectivo y preventivo.
- La evolución futura quedó limitada a funciones fuera del MVP: GPS externo, telemetría, BI, push, cloud productivo y módulos avanzados.

### Diapositiva 28: Resultados, conclusiones y cierre

- Se reorganizó el cierre en resultados, conclusiones, límites reconocidos y recomendación.
- Se mantuvieron los accesos rápidos a Figma, sistema web, evidencias app móvil y anexos.
- Se reforzó el cierre académico: NANU TECH se defiende como MVP web-móvil técnicamente delimitado, no como plataforma logística empresarial completa.

## 6. Funcionalidades fuera del MVP evitadas

No se presentaron como implementadas:

- GPS en tiempo real.
- Tracking en mapa vivo.
- Proveedor GPS externo.
- Combustible.
- Facturación.
- Mantenimiento avanzado.
- Push notifications.
- BI avanzado.
- Publicación en Play Store/App Store.
- Producción AWS estable garantizada.
- Offline robusto productivo.
- Sincronización en background completa garantizada.

Estas capacidades aparecen solo como exclusión, límite reconocido o evolución futura.
